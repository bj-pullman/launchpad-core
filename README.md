# Launchpad

> Modular operational platform for IT departments, school districts, and internal administrative teams.

---

# What is Launchpad?

Launchpad is a self-hosted modular operations platform designed to centralize internal tools, workflows, automation, reporting, and administrative systems into a unified interface.

It is built primarily for:

- K-12 technology departments
- Internal IT operations teams
- Device management teams
- Inventory and procurement workflows
- Administrative operational systems
- Organizations that prefer self-hosted infrastructure and full data ownership

Launchpad focuses on:

- Modular application architecture
- Centralized operational management
- Role-based access control (RBAC)
- Self-hosted deployment
- Operational simplicity
- Internal workflow automation
- Secure administrative tooling
- Extensible integrations

---

# Core Features

## Modular Application Platform

Launchpad supports multiple internally managed applications within a single platform experience.

Examples include:

- Finance & Budgeting
- Device Audit
- Staff Status
- User360
- SnipeOps
- Authentication & Identity
- System Maintenance
- Reporting & Operational Dashboards

---

## Identity & Authentication

Supports multiple authentication providers and hybrid identity environments.

Current/planned integrations include:

- Microsoft Entra ID
- Google Workspace
- SAML
- Local authentication
- Role/group-based authorization

---

## Asset & Inventory Operations

Launchpad integrates operational inventory workflows with systems like Snipe-IT.

Capabilities include:

- Device assignment workflows
- Audit workflows
- Room-based inventory management
- Procurement tracking
- Lifecycle reporting
- Bulk operational utilities

---

## System Maintenance

Built-in operational tooling for:

- Backups
- Rollback preparation
- Update management
- Patch notes
- Environment protection
- Hosting/service safeguards

---

## Self-Hosted Architecture

Launchpad is designed to run on infrastructure controlled by the organization.

Supported/common environments include:

- Windows Server
- Ubuntu Server
- IIS + Waitress
- NGINX reverse proxy
- Docker (planned/partial)

---

# Project Goals

Launchpad was created to solve common operational pain points caused by:

- Too many disconnected internal tools
- SaaS sprawl
- Limited automation
- Poor workflow visibility
- Fragmented inventory systems
- Difficult administrative processes

The platform aims to provide a centralized operational experience while remaining flexible and extensible.
